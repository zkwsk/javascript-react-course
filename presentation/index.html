The Internet speed run!

Brief history of the internet

  • ARPANET
    • Invented in the late 1960s
    • Developed for defense
    • Provided very simple messaging functionality
  • TCP/IP
    • Routes data between addresses
    • Foundation for the current internet
    • Can recover from package loss

HTTP

    • HTTP (Hypertext Transfer Protocol) is a communication protocol that enables the transfer of web pages and resources between a client (browser) and a server over the internet.
    • HTTP Requests:
      • GET: Get a resource
      • POST: Post data (from a form for example)
      • PUT: Update a subset of a resource
      • DELETE: Delete a resource

IP Address

  • An IP address (Internet Protocol address) is a unique numerical identifier assigned to a device on a network, enabling it to communicate with other devices over the internet or a local network.
  • Examples:
    • 192.168.1.1 (common private IP used by routers)
    • 8.8.8.8 (Google's public DNS server)

Domain Name System

  • The Domain Name System (DNS) is a decentralized system that translates human-readable domain names (e.g., example.com) into numerical IP addresses (e.g., 192.0.2.1), enabling computers to locate and communicate with each other on the internet.
  • Top-level domains
    • .com
    • .org
    • .net
    • .dk

DNS resolution

Uniform Resource Locator (URL)

  • Example: https://en.wikipedia.org/wiki/Tim_Berners-Lee
  • Protocol: http/https
  • Domain: wikipedia.org
  • Path: /wiki/Tim_Berners-Lee

Web Browsers

  • Why do we need them?
  • $ curl https://google.com
  • What does the browser do?
    • View content on the internet
    • Request web pages from servers
    • Render HTML, CSS and Javascript

Early websites

  • Example: https://info.cern.ch/
  • First website created by Tim Berners-Lee! (1991)
  • Static
  • Multiple linked pages
  • Every page written by hand
  • Default browser styling

What is a website? (1)

  • HTML - Hypertext Markup Language
    • Content "marked up" with tags that describe semantics
    • Markup for headlines, paragraphs, images, forms, etc.
  • CSS - Cascading Style Sheets (added 1996)
    • Stylistic rules explaining how certain HTML tags should be displayed
    • Rules can "cascade" in a hierarchy, so if there are multiple rules for same tag, the strongest one wins

What is a website? (2)

  • Javascript (added 1995)
  • High-level dynamic programming language
  • Used to make web pages interactive
  • Only programming language that runs in the browser
  • Since 2009 it can run on the server as well (Node.js), but you can use almost any programming language to build a web server

Dynamic websites (WEB 2.0)

  • Example: Hacker News
  • Data stored in database
  • Pages generated dynamically from the data
  • Interactive experience
  • Usually features forms that allow users or admins to input data
  • Enabled user generated content
  • Multi page applications (MPA)
  • Most logic is on server side, limited use of javascript

The render flash üí•

  • Every time you navigate to a new page, the browser downloads the HTML, CSS and JavaScript, and renders the page.
    • Flash of white screen (FOWS)
    • Flash of unstyled content (FOUC)
    • Flash of invisible content (FOIC)
  • The website might forget things when you change pages
  • Not a seamless user-experience
  • Not good for "rich" interactive experiences, like games

Rich internet applications

  • Applications that are downloaded and run in the browser
  • Java applets
    • Written in Java
    • Discontinued from 2013 and onwards
    • Often used for early corporate web applications
    • Banking applications
    • Often the UI would not feel at home in the browser and would feel more like a Windows application.

Java applets examples

Rich internet applications

  • Flash applets
    • Written in Actionscript
    • Discontinued from 2017 and onwards
    • Apple never allowed flash to run on iOS
    • Rich multimedia applications (i.e. video players)
    • Infamous for banner advertising and pop-ups

Native applications (1)

  • Desktop applications (Windows, Mac, Linux)
  • Mobile applications (iOS, Android)
  • Written in a "native" language:
    • iOS: Swift or Objective-C
    • Android: Java
    • Windows: C#, Java, etc.
    • Mac: Swift, Objective-C, Java, etc.

Native applications (2)

  • Must be downloaded in full to run
  • Updates requires downloading the app again
  • Pain to manage updates in large organisations
  • Works while offline
  • Very fast once downloaded, can tap into all resources of device (i.e. graphics processor, multimedia, etc.)

Rich web applications

  • Browsers are gaining more "rich" features:
    • Multimedia playback
    • Access to camera
    • Access to GPS
    • Offline features
    • Graphics acceleration
  • Web developers have found ways to make web applications feel more like native applications

Single Page Applications (SPA)

  • Making the experience of web applications seamless
  • No more render flashes
  • Application can remember complex state in between pages
  • Examples: Youtube, Facebook, Reddit

SPAs hiding in plain sight

  • Many apps you download to your computer or phone are web applications that are packaged in a "thin" invisible web browser
  • Examples: Spotify, Visual Studio Code, Instagram
  • Companies that offer a full web-based experience often prefer to use the same code-base and the same developers for the web app and for the desktop and mobile apps
  • These apps run slower than native counterpars written in a platform-specific language

How Does a SPA Work? (1)

  • Only one (empty) HTML document
  • After initial page is loaded, javascript will fetch content asynchronously (in an AJAX request)
  • When a user click a link, the app send a request to fetch only the relevant content. The app will modify the existing HTML and inject content without reloading the page
  • If only a small part of the UI is updated, only that data is downloaded.
  • Since there is no reload any settings the user has made will not be forgotten.

How Does a SPA Work? (2)

  • In a MPA you can only retrieve a new page by modifying the URL. In a SPA it is an optional step that makes the app work more like a traditional website. This has multiple benefits:
    • Deep linking
    • Search Engine Optimization

Drawbacks to the SPA architecture

  • In a SPA all the application code runs in the users browser:
    • First load can be slow, because a large javascript bundle is downloaded
    • There can be security concerns because client code be viewed by all users

Hybrid architectures

  • Practices from SPA and MPA are combined to achieve the best of both worlds:
    • On initial request, instead of waiting for javascript to load, you might use the server to render an initial state of the app, which subsequently will run as a SPA
    • Some apps are predominantly MPA for legacy- or security reasons. But selected parts of the app might work like a SPA to provide a more "rich" interactive experience

Why React

  • React is a library that allow you to build UI components
  • Designed for SPA use
  • React is not "opinionated" and can be configured to work together with many other technologies
  • React works great rendered on the client as well as on the server
  • React powers large apps such as Facebook, Instagram and many, many more

In summary

  • Internet evolved from static documents, to dynamic apps with limited user experience into rich interactive apps that can no longer be discerned from native apps
  • The SPA architecture is one of the main contributors to this shift
  • MPA and SPA architectures can often be combined and tailored to yield even better performance or experiences
  • React is very popular framework for building SPAs

Learn to format content for the web

Learning goals

  • Understand the purpose of writing HTML

HTML Basics


  • HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of web pages using markup.
  • HTML elements are the building blocks of HTML pages. With HTML constructs, images and other objects, such as interactive forms, may be embedded into the rendered page.
  • HTML adds semantics to your content

Why do we need HTML?


  • Without HTML tags the browser would have no way to know what to do with your content.
    • All text would be displayed the same
    • Instead of rendering an image the URL would be displayed
    • Tabular data would be one big messy block of text

The HTML tag


Below is a "p" tag which tells the browser that the content within is a paragraph of text: 

                    

<p>This is a paragraph</p>

The tag itself is not displayed to the user, only the content. But the browser uses the tag to infer how to display the content. In this case the browser will add a line break after each paragrap of text to segment it from the next paragraph.

Other examples


                    
<h1>This is a heading</h2>
<h2>This is a slightly smaller heading</h2>
<img src="https://picsum.photos/200/300" />

Opening and closing tags


HTML elements usually consist of an opening tag, some content and a closing tag:

                    

<p>Content is here</p>

But in some cases the tag does not have any content and a self-enclosing tag can be used:

                    

<hr/>

Nesting HTML tags


Besides raw content, HTML might have tags nested inside other tags:

                    

<ul>
    <li>Item 1</li>
    <li>Item 2
        <ul>
        <li>Sub-item 1</li>
        <li>Sub-item 2</li>
        </ul>
    </li>
    <li>Item 3</li>
</ul>

HTML indentation


We usually indent the tags that are nested, to improve readability, but it is not a requirement. These snippets are all equivalent: 

                    

<ul>
    <li>Item</li>
</ul>

<ul>
<li>Item</li>
</ul>

<ul><li>Item</li></ul>

Generic HTML tags


Some tags exist only to impart structure and has no semantic meaning:

                    

<div>Some content</div>
<span>Some content</span>
  • These tags are often used to group content together for styling purposes in CSS. 
  • div
    is block level and will always break into new line 
  • span
    is inline level and can be used inside block level tags (paragraph for example)

Semantic HTML tags


  • Some tags have a specific semantic meaning and are used to describe the content
  • Usually they do not affect the visual styling, but they can be customized as needed
                    

<header>Header content</header>
<footer>Footer content</footer>
<nav>Navigation content</nav>
<article>Article content</article>
<section>Section content</section>
<aside>Aside content</aside>
<main>Main content</main>

HTML attributes


HTML tags may have attributes describing them further. For example the image tag uses the 

src
attribute to distinguish the url of the image to be displayed:

                    

<img src="https://picsum.photos/200/300" />

The 

src
attribute is specific to the image tag and cannot be used on other tags.

HTML class attributes


Some attributes can be used on any tag. For example the 

class
attribute can be used to add a class to any tag:

                    

<p class="paragraph-class">Some content</p>

Classes are typically used to group instances of the same tag into classes so you can apply custom styling in CSS to display them different from other classes.

HTML class examples


You might have a rule in CSS defining that all paragraphs are in black text, but what if you want certain text to be blue or red? You can add a class:

                    

<p class="blue">Some blue text</p>
<p class="red">Some red text</p>
  • Note that you cannot do this in HTML alone. You will need to add corresponsing styling rules in CSS, which we will cover later.
  • The main takeaway is that classes allow HTML and CSS to communicate together

HTML ID attributes


  • IDs are used to identify a specific element in the document.
  • Classes can be applied to many elements, IDs should only be applied once in the document.
  • While classes are often used to style groups of elements, IDs are often used by javascript to uniquely target an element operate on.
  • If you link to a specific ID in a page, the browser will scroll that element into view when the link is clicked.

HTML ID examples


                    

<h1 id="main-heading">Main heading</h1>
<p id="paragraph-1">Some content</p>
<p id="paragraph-2">More content</p>

You can combine classes and IDs:

                    

<p class="blue" id="paragraph-1">Some blue text</p>
<p class="red" id="paragraph-2">Some red text</p>
<p class="blue" id="paragraph-3">More blue text</p>

CSS Introduction

Cascading Style Sheets


  • CSS (Cascading Style Sheets) is a style sheet language used for describing the presentation of a document written in HTML
  • CSS describes how elements should be rendered on screen, on paper, in speech, or on other media

Why do we need CSS?


  • Browsers apply default styling to web pages
  • If we just want to create generic documents we do not need CSS
  • Without CSS, web pages would all have the same simple style and look plain and boring
  • CSS allow you to layout the page so it looks more like a modern app and less like a webpage from the 90s

Scope


  • Learning CSS is simple at the surface level, but very complicated practice
  • The API is very comprehensive
  • Browser might work different from one another
  • There are UI libraries available that reduces the need to know CSS in depth
  • Scope of the class is to learn the basic mechanics of CSS and how it interact with HTML. Not to memorise a lot of properties and selectors.

Where to put CSS?

  • Directly on individual HTML element
  • Embedded in the 
    head
    tag of the page
  • In a separate linked .css file

Inline CSS


Example:

                    
<p style="color: blue; font-size: 20px;">This is a paragraph</p>

  • Inline CSS is not recommended because it mixes content with presentation
  • You would have to repeat styling on every tag
  • HTML becomes hard to maintain

CSS in style tag


  • Rule applies to all p tags on page
  • You need to repeat the style tag on every page
                    
<!DOCTYPE html>
<html lang="en">
<head>

            font-size: 20px;
        }
    </style>
</head>
<body>
    <p>This is a paragraph</p>
</body>
</html>

    link
    tag
  • The 
    link

    head
    tag
  • Many pages can link the same external css file ensuring consistent styling and less duplicated code
  • This is the recommended practice

A general CSS rule

  • Selector: Which element(s) do you want to style?
  • Property: What do you want to change?
  • Value: The new value of the property 

                    
selector { property: value; }

Selector


                    
selector { ... }

  • Select HTML elements
  • Select custom classes
  • Select elements based on ID (not recommended)
  • Advanced selection
  • Full reference: https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_selectors

The element selector


  • The element selector selects elements based on the element name
  • Select all 
    p
    tags: 
                    
p {
    property: value;
}

The CSS class selector


  • The class selector is used to select elements with a specific class attribute
  • The class selector is defined with a period:
                    
.classname {
    property: value;
}

.blue-text {
    color: blue;
}

Combining selectors (1)


  • You can combine selectors to target specific elements
  • "Select all p tags with the class .blue":

    color: blue;
}

<p class="blue">This is a blue paragraph</p>

Combining selectors (2)


  • "Select all p tags nested within an element with the class .blue":

                    
.blue p {
    color: blue;
}

<div class="blue">
    <p>This is a blue paragraph</p>
</div>

CSS properties and HTML elements

  • The properties available to style an element depends on the element.
  • Generally all block level elements have width and height properties and inline elements do not
  • Some elements have special properties. For example buttons and anchor tags have the ability to customise the mouse cursor on hover, while this is not possible on most other elements.

CSS in JS


  • In modern web development it is common to adopt an approach where CSS is automatically added directly to the individual HTML tags with Javascript.
  • This is not in violation with our need to ensure consistency because styling logic is centralised in javascript instead of being centralised in a CSS file.
  • Can be more flexible when javascript frequently needs to modify CSS.

Summary

  • CSS is used to style how HTML elements are displayed on screen
  • A CSS rule constists of a selector, property and value
  • Different selectors can be combined to refine the selection
  • If no custom rules are applied the browser default rules will be used
  • It is recommended to keep CSS in a separate file and link it into the HTML document

ES6+ features

Array iterator methods


  • A lot of Javascript and particularly React is written in a function oriented manner
  • Array iterator methods are a set of functions that operate on arrays and return new arrays
  • They are a little more "declarative" than for loops in that they give a bit more context to what the code is doing.
  • A for a while loop is generic and you would have to study the code to understand what it does.

Array.map()


Takes in an array and creates a new array by transforming each element. 

                    
const array1 = [1, 4, 9, 16];

// Pass a function to map
const map1 = array1.map((x) => x * 2);

console.log(map1);
// Expected output: Array [2, 8, 18, 32]

Array.filter()


Creates a new array with only elements that pass a test.

                    
const words = ["spray", "elite", "exuberant", "destruction", "present"];

const result = words.filter((word) => word.length > 6);

console.log(result);
// Expected output: Array ["exuberant", "destruction", "present"]

Other array iterator functions


  • Array.find() - works like filter, but only returns first matching element
  • Array.some() - returns true if at least one element passes a given test.
  • Array.reduce() - reduces an array to a single value. Could for example be used to sum all values.

Spread operator ...


Used to expand an array or object into individual elements. 

                                              
const arr1 = [1, 2, 3];

const obj1 = { a: 1, b: 2 };
const obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }

Destructuring assignment


Extract values from objects and arrays.
