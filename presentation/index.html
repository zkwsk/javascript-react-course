Brief history of the internet

  • ARPANET
    • Invented in the late 1960s
    • Developed for defense
    • Provided very simple messaging functionality
  • TCP/IP
    • Routes data between addresses
    • Foundation for the current internet
    • Can recover from package loss

HTTP

    • HTTP (Hypertext Transfer Protocol) is a communication protocol that enables the transfer of web pages and resources between a client (browser) and a server over the internet.
    • HTTP Requests:
      • GET: Get a resource
      • POST: Post data (from a form for example)
      • PUT: Update a subset of a resource
      • DELETE: Delete a resource

IP Address

  • An IP address (Internet Protocol address) is a unique numerical identifier assigned to a device on a network, enabling it to communicate with other devices over the internet or a local network.
  • Examples:
    • 192.168.1.1 (common private IP used by routers)
    • 8.8.8.8 (Google's public DNS server)

Domain Name System

  • The Domain Name System (DNS) is a decentralized system that translates human-readable domain names (e.g., example.com) into numerical IP addresses (e.g., 192.0.2.1), enabling computers to locate and communicate with each other on the internet.
  • Top-level domains
    • .com
    • .org
    • .net
    • .dk

DNS resolution

Uniform Resource Locator (URL)

  • Example: https://en.wikipedia.org/wiki/Tim_Berners-Lee
  • Protocol: http/https
  • Domain: wikipedia.org
  • Path: /wiki/Tim_Berners-Lee

Early websites

  • Example: https://info.cern.ch/
  • First website created by Tim Berners-Lee! (1991)

  • Multiple linked pages
  • Every page written by hand
  • Default browser styling

What is a website? (1)

  • HTML - Hypertext Markup Language
    • Content "marked up" with tags that describe semantics
    • Markup for headlines, paragraphs, images, forms, etc.
  • CSS - Cascading Style Sheets (added 1996)
    • Stylistic rules explaining how certain HTML tags should be displayed
    • Rules can "cascade" in a hierarchy, so if there are multiple rules for same tag, the strongest one wins

What is a website? (2)

  • Javascript (added 1995)
  • High-level dynamic programming language
  • Used to make web pages interactive
  • Only programming language that runs in the browser
  • Since 2009 it can run on the server as well (Node.js), but you can use almost any programming language to build a web server

Dynamic websites (WEB 2.0)

  • Example: Hacker News
  • Data stored in database
  • Pages generated dynamically from the data
  • Interactive experience
  • Usually features forms that allow users or admins to input data
  • Enabled user generated content
  • Multi page applications (MPA)
  • Most logic is on server side, limited use of javascript

    • Flash of white screen (FOWS)

Rich internet applications

  • Applications that are downloaded and run in the browser
  • Java applets
    • Written in Java
    • Discontinued from 2013 and onwards
    • Often used for early corporate web applications
    • Banking applications
    • Often the UI would not feel at home in the browser and would feel more like a Windows application.

Java applets examples

Rich internet applications

  • Flash applets
    • Written in Actionscript
    • Discontinued from 2017 and onwards
    • Apple never allowed flash to run on iOS
    • Rich multimedia applications (i.e. video players)
    • Infamous for banner advertising and pop-ups

Native applications (1)

  • Desktop applications (Windows, Mac, Linux)
  • Mobile applications (iOS, Android)
  • Written in a "native" language:
    • iOS: Swift or Objective-C
    • Android: Java
    • Windows: C#, Java, etc.
    • Mac: Swift, Objective-C, Java, etc.

Native applications (2)

  • Must be downloaded in full to run
  • Updates requires downloading the app again
  • Pain to manage updates in large organisations
  • Works while offline
  • Very fast once downloaded, can tap into all resources of device (i.e. graphics processor, multimedia, etc.)

Rich web applications

  • Browsers are gaining more "rich" features:
    • Multimedia playback
    • Access to camera
    • Access to GPS
    • Offline features
    • Graphics acceleration
  • Web developers have found ways to make web applications feel more like native applications

Single Page Applications (SPA)

  • Making the experience of web applications seamless
  • No more render flashes
  • Application can remember complex state in between pages
  • Examples: Youtube, Facebook, Reddit

SPAs hiding in plain sight

  • Many apps you download to your computer or phone are web applications that are packaged in a "thin" invisible web browser
  • Examples: Spotify, Visual Studio Code, Instagram
  • Companies that offer a full web-based experience often prefer to use the same code-base and the same developers for the web app and for the desktop and mobile apps
  • These apps run slower than native counterpars written in a platform-specific language

How Does a SPA Work? (1)

  • Only one (empty) HTML document
  • After initial page is loaded, javascript will fetch content asynchronously (in an AJAX request)
  • When a user click a link, the app send a request to fetch only the relevant content. The app will modify the existing HTML and inject content without reloading the page
  • If only a small part of the UI is updated, only that data is downloaded.
  • Since there is no reload any settings the user has made will not be forgotten.

How Does a SPA Work? (2)

  • In a MPA you can only retrieve a new page by modifying the URL. In a SPA it is an optional step that makes the app work more like a traditional website. This has multiple benefits:
    • Deep linking
    • Search Engine Optimization

Drawbacks to the SPA architecture

  • In a SPA all the application code runs in the users browser:
    • First load can be slow, because a large javascript bundle is downloaded
    • There can be security concerns because client code be viewed by all users

Hybrid architectures

  • Practices from SPA and MPA are combined to achieve the best of both worlds:
    • On initial request, instead of waiting for javascript to load, you might use the server to render an initial state of the app, which subsequently will run as a SPA
    • Some apps are predominantly MPA for legacy- or security reasons. But selected parts of the app might work like a SPA to provide a more "rich" interactive experience

Why React

  • React is a library that allow you to build UI components
  • Designed for SPA use
  • React is not "opinionated" and can be configured to work together with many other technologies
  • React works great rendered on the client as well as on the server
  • React powers large apps such as Facebook, Instagram and many, many more

In summary

  • Internet evolved from static documents, to dynamic apps with limited user experience into rich interactive apps that can no longer be discerned from native apps
  • The SPA architecture is one of the main contributors to this shift
  • MPA and SPA architectures can often be combined and tailored to yield even better performance or experiences
  • React is very popular framework for building SPAs

Learning Bash Command Line


Get ready to learn the essential bash commands for managing files, directories, and working with Node.js and Git!

Learning goals

  • Understand how to navigate between directories
  • Understand to view the contents of a directory
  • Understand how to run scripts
  • Understand how to set up a Node.js project
  • Understand how to add dependencies with NPM
  • Understand how to clone a project with Git

Check Node.js & Git Versions

Check Node.js version:

node --version

Example output:

v20.10.0

Check Git version:

git --version

Example output:

git version 2.48.1

Navigate Between Directories


Use the cd command to navigate between directories in the file system.

cd /path/to/directory

Examples:

cd Documents
cd .. (to go up one directory level)
cd ~ (to go to the home directory)

See the current directory path


Use the pwd command to display the path to the current directory you are working in:

pwd

Example output:

/user/home

View the Contents of a Directory


Use the ls command to list files and directories within the current directory.

ls

Examples:

ls -l (long listing format)
ls -a (include hidden files)
ls /path/to/directory (list contents of another directory)

Create a New Directory


Use the mkdir command to create a new directory. 

mkdir new_directory

Examples:

mkdir my_project (creates a directory named 'my_project')
mkdir -p parent/child (creates nested directories)

Setup a Node.js Project


To create a Node.js project, initialize it with npm init. 

npm init

Follow the prompts or use npm init -y for default values.

Example:

npm init -y (automatically creates a package.json file)

Add Dependencies with NPM


Use npm install to install dependencies for your project.

npm install

Examples:

npm install express (installs Express.js)
npm install --save-dev  (installs as a devDependency)

Run an NPM Script


Use npm run followed by the script name to execute an NPM script defined in package.json. 

npm run script-name

Example:

npm run start (runs the 'start' script)
npm run test (runs the 'test' script)

Check available scripts in package.json under the "scripts" section:


      {
        "scripts": {
          "start": "node app.js",
          "test": "jest"
        }
      }

6. Clone a Project with Git

Use git clone to clone a repository from GitHub or another Git hosting service.

git clone

Example:

git clone https://github.com/username/project-name.git
